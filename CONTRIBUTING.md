# Contributing to the BOSC Community Library

First off, thank you for considering contributing to the BOSC Community Library. It's people like you that make it such a great tool for the public sector.

## Development Workflow

1.  **Fork** the repository on GitHub.
2.  **Clone** your fork to your local machine.
3.  Ensure your environment matches the requirements in `package.json` (Node.js v18+).
4.  Install dependencies: `npm install`
5.  Create a **new branch** for your feature or bug fix: `git checkout -b feature/your-feature-name` or `bugfix/issue-number`
6.  Make your changes and test them locally: `npm run dev`
7.  Commit your changes following the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) standard.
8.  Push to your fork: `git push origin your-branch-name`
9.  Submit a **Pull Request**, filling out the provided template.

## Technical Expectations

*   **TypeScript:** All new code must be written in TypeScript with strict typing.
*   **Styling:** We use Tailwind CSS. Please adhere to the existing responsive design principles.
*   **Documentation:** Update the `README.md` and inline JSDoc comments for any significant new architecture components.

## Issue Triage

We label issues to help you find where you can best contribute:
*   `good first issue`: Ideal for newcomers.
*   `help wanted`: The core team needs assistance on this.
*   `bug`: Confirmed bugs in the system.
*   `enhancement`: New features or improvements.

Thanks for participating in our open source community!
